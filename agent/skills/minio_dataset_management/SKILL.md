# MinIO Dataset Management

You must never store massive gigabyte-scale `.csv` files or `.pt` weight files statically in the `/experiments` host volume, as it inflates disk space rapidly. Use the MinIO S3 deployment instead to handle object storage.

## Access Details
- **Endpoint**: `http://127.0.0.1:9000` (Routed via Consul Connect)
- **Library**: Use the `boto3` library inside your virtual environments.
- **Root Credentials**: You can read the raw access keys generated by the Vault template if you need to authenticate `boto3` directly. They are mapped into the environment configuration of the cluster components.

## Upload / Download Methodology

Prefer programmatic uploads and downloads rather than downloading files using `wget` to your local SSD.

```python
import boto3
from botocore.client import Config

s3 = boto3.client('s3',
                  endpoint_url='http://127.0.0.1:9000',
                  aws_access_key_id='minioadmin', # Assuming default test setup, check ENV otherwise
                  aws_secret_access_key='minioadmin',
                  config=Config(signature_version='s3v4'),
                  region_name='us-east-1')

def download_dataset():
    # Downloads a CSV required for an experiment
    s3.download_file('datasets', 'medical_records.csv', '/experiments/temp_medical_records.csv')

def upload_weights(local_path):
    # Uploads a massive finished model to cold storage
    s3.upload_file(local_path, 'models', 'nanobot_v1_weights.pt')
```
